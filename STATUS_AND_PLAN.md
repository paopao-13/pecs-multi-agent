## 2. 路线 B 行动清单（按优先级）

### ✅ Step 1 — 换 API 后跑 P0 bug 回归（必做，验证修复）
- 改 `.env`：`LLM_API_KEY` / `LLM_BASE_URL` / `LLM_MODEL` 指向新 API

- 重点确认：报错文本不再被当答案冻结（P0 修复生效）
- 不限流时可 `set LLM_MIN_GAP=0` 关掉节流提速
- 产物：`results/_resume_*.json` 进度 + stdout 的 DONE/CORRECT

### ✅ Step 2 — WebShop 明确降级标注（必做，消除误导）
- 在 `tools/webshop.py` 顶部 docstring 已写明 "Local WebShop-style ... adapter"，
  需在**报告/README**里显式声明：
  > "WebShop 模块为本地适配器（8 商品约束匹配），非 AgentBench 真实环境，
  > 成功率不计入 +18pp 目标，仅用于演示约束解析流程。"
- 不写真实 WebShop 集成代码（路线 B 不做）

### ✅ Step 3 — GAIA 诚实标注（必做，消除"96% 灌水"误导）
- 在报告里拆分两类题：
  - **确定性计算题**（15/28，启发式 0-token 秒杀）—— 说明这是框架的"免费得分"，不代表推理能力
  - **知识+推理题**（约 13/28）—— 这才是框架真实优势所在，对比 ReAct 有 +20pp
- 明确写："当前为 33 道内置样本（偏计算），非官方 165 题 GAIA L1 分布；
  指标仅作演示，不代表官方榜单成绩"

### ✅ Step 4 — Token 指标重述（必做，消除"38% 假象"）
- 报告中区分两个数字：
  - **机制本身（预算调度消融）：−4.5%**（诚实值，承认未达 −30%）
  - **端到端 vs 失控 ReAct：−38.78%**（说明这是修复 ReAct 上下文雪崩的副产品，非调度单独贡献）
- 不硬凑 −30%；若要接近，需后续落地"角色合并 + 模型路由"（路线 B 可选，非必须）
