# 产品需求文档 (PRD)

> pecs-multi-agent 项目需求分析，包括问题背景、目标用户、功能需求和非功能需求。

## 1. 问题背景

### 1.1 现状痛点

传统单 Agent 系统（如 ReAct）在处理复杂任务时存在三个核心问题：

| 问题 | 具体表现 | 影响 |
|------|----------|------|
| **质量不稳定** | 单个 LLM 同时承担规划、执行、检查，职责混淆 | 复杂任务（大数计算、多步推理）准确率下降 |
| **成本不可控** | 反复调用 LLM 直到任务完成，无预算上限 | 简单任务和复杂任务成本差异巨大，难以预估 |
| **缺乏纠错** | 出错后无专门角色评审和反馈 | 错误从执行阶段一路传递到最终答案 |

### 1.2 解决方案

设计 PECS 四角色多智能体协作框架：
- **P**lanner：任务拆解，生成步骤列表
- **E**xecutor：调用工具执行子任务
- **C**ritic：三维评分（准确/一致/完整），拦截错误
- **S**ynthesizer：综合所有结果生成最终答案

通过 LangGraph StateGraph 串联，形成 Plan-Execute-Reflect 闭环，配合 Token 预算感知调度实现成本可控。

## 2. 目标用户

## 3. 功能需求

### 3.1 核心功能（P0）

| 需求ID | 功能 | 验收标准 |
|--------|------|----------|
| F-01 | PECS 四角色协作 | Planner→Executor→Critic→Synthesizer 完整流转，支持反思循环 |
| F-02 | Token 预算感知调度 | 50000 Token 上限，70%/85%/95% 三级降级，单任务成本可控 |
| F-03 | AST 安全沙箱 | 拦截 import/exec/eval/open 等 16 种危险操作，两层防护 |
| F-04 | GAIA 评测 | 28 题样例集准确率 ≥ 75%，对比 ReAct 基线提升 ≥ 15pp |
| F-05 | ReAct 基线对照 | 同模型同工具同题目公平对比 |
| F-06 | 消融实验 | 4 组完全移除型 + 2 组功能关闭型，单一变量控制 |

## 4. 非功能需求

| 维度 | 要求 | 实现方式 |
|------|------|----------|
| **安全性** | LLM 生成代码安全执行 | AST 预检查 + 白名单沙箱双层防护 |
| **可配置性** | 模型/预算/阈值可配置 | YAML + 环境变量 + 硬编码三级配置 |
| **可测试性** | 核心逻辑单元测试覆盖 | 29 项 pytest 测试，无外部依赖 |
| **可扩展性** | 支持自定义 Agent 和工具 | 继承原生角色 + 工具注册 |
| **可移植性** | 跨平台运行 | Python 3.10+，Windows/macOS/Linux |
| **成本可控** | 单任务 Token 有上限 | 50000 Token 预算 + 三级降级 |
| **类型安全** | 状态字段运行时校验 | Pydantic BaseModel + 字典兼容 |

## 5. 约束条件

1. **模型约束**：使用 DeepSeek-V3 API（成本约为 GPT-4 的 1/10），通过 OpenAI 兼容接口调用
2. **数据约束**：评测使用内置 28+6 道 Mock 样例，非 GAIA 官方 466 题测试集
3. **资源约束**：学生项目，单机部署，无 GPU 需求
4. **开发约束**：个人独立开发，使用 AI 编程辅助工具加速编码

## 6. 验收标准汇总

| 指标 | 目标值 | 实测值 | 达标 |
|------|--------|--------|:---:|
| GAIA L1 准确率 | ≥ 75% | 100% (28/28) | ✅ |
| vs ReAct 准确率提升 | ≥ 15pp | +21.4pp | ✅ |
| WebShop 成功率提升 | ≥ 18pp | +33.3pp | ✅ |
| Token/任务降低 | ≥ 30% | 86.8% | ✅ |

## 7. 风险与缓解

| 风险 | 概率 | 影响 | 缓解措施 |
|------|:---:|:---:|----------|
| Mock 数据偏乐观 | 高 | 中 | 文档诚实声明 + 官方数据集接入就绪 |
| 启发式覆盖有限 | 中 | 中 | 标注为已知问题 + embedding 缓存优化方向 |
| LLM API 不可用 | 低 | 高 | 启发式兜底层保证基本可用 |
| AST 沙箱被绕过 | 低 | 高 | 双层防护 + 持续更新黑名单 |
