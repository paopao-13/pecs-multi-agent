# 代码评审流程

> 本项目为个人独立开发项目，采用简化的自评审流程，确保代码质量。

## 1. 项目性质声明

- **开发模式**：个人独立开发
- **辅助工具**：使用 AI 编程辅助工具（Codex / Claude Code）加速编码
- **技术决策**：所有架构设计和选型决策由开发者本人完成
- **代码审查**：采用自评审 + AI 辅助审查模式

## 2. 自评审检查清单

每次提交前，按以下清单逐项检查：

### 2.1 代码质量

- [ ] 代码通过 `pytest tests/ -v` 全部测试
- [ ] 代码通过 `black --check .` 格式检查
- [ ] 代码通过 `isort --check-only .` 导入排序检查
- [ ] 代码通过 `flake8 . --select=E9,F63,F7,F82` 语法检查
- [ ] 新增函数有 docstring 说明
- [ ] 无调试残留代码（print / breakpoint / TODO）

### 2.2 安全性

- [ ] 新增工具通过 AST 安全沙箱检查
- [ ] 无硬编码 API Key / 密码 / 私钥
- [ ] `.env` 文件在 `.gitignore` 中
- [ ] 新增配置项同步更新 `.env.example`

### 2.3 文档同步

- [ ] 修改功能时同步更新 README
- [ ] 新增模块时更新项目结构说明
- [ ] 修改接口时更新 API 文档
- [ ] 修复缺陷时更新 CHANGELOG

### 2.4 实验数据

- [ ] 评测数据来自真实运行（非手工编造）
- [ ] Mock 数据标注"非官方测试集"
- [ ] 消融实验保留单变量控制
- [ ] Token 统计为端到端对比（含 LLM + 工具全流程）

## 3. AI 辅助审查

使用 AI 编程工具进行代码审查，关注以下维度：

| 维度 | 审查重点 |
|------|----------|
| 逻辑正确性 | 条件分支覆盖、边界情况处理 |
| 安全性 | 输入验证、沙箱完整性、注入风险 |
| 性能 | 不必要的 LLM 调用、重复计算、内存泄漏 |
| 可维护性 | 命名清晰、函数职责单一、模块边界明确 |

## 4. 提交规范

遵循 Conventional Commits 规范：

```
<type>: <description>

<body>
```

| Type | 用途 | 示例 |
|------|------|------|
| `feat` | 新功能 | `feat: 添加异步 LLM 调用支持` |
| `fix` | 缺陷修复 | `fix: 修复沙箱无超时问题` |
| `docs` | 文档更新 | `docs: 添加部署方案文档` |
| `chore` | 杂项 | `chore: 升级 langgraph 版本` |
